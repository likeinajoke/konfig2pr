# Минимальное настраиваемое CLI-приложение

Пользователь настраивает такие параметры, как: имя анализируемого пакета, URL-адрес репозитория или путь к файлу тестового репозитория, режим работы с тестовым репозиторием, максимальная глубина анализа зависимостей.
Для этого была реализована функция parse_arguments, с помощью которой мы добавляем аргументы для работы в командой строке: --package, --repository, --mode, --depth, соответсвенно.

Были реализованы следующие функции для проверок корректности вводимых пользоваетелем значений аргументов:

1. validate_package_name - проверка корректности имени пакета, которое должно быть непустой строкой.
2. validate_repository - проверка корректность URL репозитория или пути к файлу
3. validate_mode - проверка корректности глубины анализа
4. validate_depth - проверка корректности глубины анализа

В главной функции реализован вывод всех параметров, настраиваемых пользователем.

Реализован 21 тест с помощью пакета Pytest в отдельном файле test_main.py.
